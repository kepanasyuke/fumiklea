🐧 Линус Торвальдс и новая Java 25: Путь к простоте
Новая версия Java 25 стала не просто очередным релизом, а своеобразным мостом, соединившим философии двух, казалось бы, разных миров. В мире Java произошли изменения, которые, несомненно, нашли бы отклик у создателя Linux — Линуса Торвальдса.

https://www.linuxfoundation.org/hubfs/LinusTorvalds.jpeg

Линус Торвальдс — финско-американский разработчик, создатель ядра Linux и системы контроля версий Git. Он известен своим прагматизмом и критикой излишне сложных решений в программировании. Он часто высказывался о Java как о "ужасном языке" из-за её многословия и неоправданной сложности для системного программирования.

☕ Почему Java 25 «поменяла курс»?
В последнее время в языке произошли существенные изменения. JDK 25 представил нововведения, которые делают Java более простой и гибкой, сближая её с миром C в некоторых важных аспектах.

Ближе к «железу»: Появился Foreign Function & Memory (FFM) API — механизм, который позволяет вызывать нативный код на C из Java без использования JNI. Это даёт Java-разработчикам уровень контроля и производительности, который раньше был доступен только C/C++ программистам.

Язык для всех: Новые «Compact Source Files» позволяют писать простые скрипты и утилиты без шаблонного кода (отпала необходимость в объявлении класса и статического метода main). Это делает Java значительно проще для входа, приближая её к философии «напиши и запусти».

Эти два направления кажутся противоположными, но они объединены одной целью: дать разработчику больше возможностей, когда это нужно, и не мешать, когда это не требуется.

🚀 Так бы сказал Линус
Увидев эти изменения, Линус, вероятно, не стал бы называть Java «ужасной», а скорее сказал бы что-то вроде:

«О, наконец-то. Раньше Java напоминала мне те ужасные проекты, где для простой задачи нужно было заполнить десять страниц документации и трижды поклониться архитектору. Новый FFM — это как получить прямой доступ к оборудованию, минуя слой кривого менеджмента. А компактные скрипты... ну, наконец-то вы поняли: иногда лучшее, что можно сделать — это просто не мешать разработчику. Это уже больше похоже на инженерный подход, а не на бюрократический цирк».*

# 💾 FreaXExplorer v1.91 — Ретро-терминал археологии Unix & Linux


Элегантное десктопное приложение на **Java Swing**, стилизованное под монохромный зеленый CRT-монитор (люминофор P1, IBM 5151). Программа имитирует низкоуровневую «археологическую» выгрузку исторических артефактов Unix/Linux из архивов **TUHS (The Unix Heritage Society)**.

---

## 🛠️ Основные фичи

* **Симуляция дискеты 3.5"**: Пошаговое чтение магнитных дорожек (CHS-адресация) с аналоговыми задержками и случайными сбоями секторов (CRC Error).
* **Электронный PC Speaker**: Интеграция звукового отклика (`Toolkit.beep()`) при сбоях чтения и подтверждении операций.
* **Исторические пасхалки**: Реальные комментарии Линуса Торвальдса из исходников ядер `v0.01-0.11` и хроники дебатов с Эндрю Таненбаумом.
* **Глобальный Key Dispatcher**: Железобетонный перехват горячих клавиш, полностью защищенный от блокировок текстового поля.

---

## 🎮 Управление (Горячие клавиши)

* **`[F5]`** — Запустить механическую эмуляцию чтения дискеты с архивом.
* **`[F8]`** — Открыть оригинальный веб-сайт сообщества **TUHS** в браузере.
* **`[Y / N]`** — Интерактивный ответ на запрос перенаправления в конце симуляции.
* **`f-r-e-a-x`** — Секретный пасхальный код. Наберите его посимвольно в любой момент, чтобы вызвать покадровую ASCII-анимацию «моргающего» пингвина **Tux** и узнать историю кодового имени ОС.

---

## 🛑 Почему Makefile в Codespaces сейчас бесполезен?

- **Отсутствие дисплея:** Java Swing создавалась для домашних ПК. На серверах GitHub нет видеокарты и монитора, поэтому Java пытается подключиться к «воздуху».
- **Блокировки туннелей:** Утилита `localtunnel` часто режет веб-сокеты и бинарные потоки noVNC. Вместо окна программы вы видите белый экран, ошибку 404 или список папок `vnc.html`.
- **Зачем усложнять?** Чтобы запустить Java Swing в браузере «через дивы на JS», пришлось бы переписывать весь графический интерфейс с нуля на веб-технологии. Но есть готовый код на Java, который идеально приспособлен для запуска на вашем собственном компьютере!

## 🚀 Вариант 1 (Идеальный): Запуск в IntelliJ IDEA (в 1 клик)

Это самый надёжный способ. Всё, что вам нужно — перенести текстовый файл с кодом на ваш компьютер.

### Пошаговый запуск:

1. **Откройте IntelliJ IDEA** на своём компьютере.
2. **Создайте новый проект:** `File → New → Project` (выберите обычный **Java Project**).
3. В левой панели проекта найдите синюю папку `src`. Нажмите на неё правой кнопкой мыши и выберите `New → Java Class`.
4. **Назовите класс** строго `Main` (или `FreaXExplorer` — в зависимости от того, как назван `public class` в вашем коде).
5. **Скопируйте** весь последний Java-код и вставьте его внутрь созданного класса, полностью заменив шаблонный текст.
6. **Запуск:** в самом верху IntelliJ IDEA нажмите на зелёный треугольник (кнопку **Run**) или используйте сочетание клавиш `Shift + F10`.

### Результат:

На вашем рабочем столе мгновенно откроется аккуратное ретро-окно `640×480` с мигающим зелёным люминофором. Клавиатура будет активна сразу:
- Нажмите **F5** — пойдёт симуляция дискеты со звуком мотора и писком ошибок.
- Введите слово **freax** — Tux замашет крыльями без каких-либо задержек!
