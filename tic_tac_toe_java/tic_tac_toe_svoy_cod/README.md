# ⭕ Крестики-нолики (Tic‑Tac‑Toe) на JavaFX

**Версия**: 1.0-SNAPSHOT  
**Статус**: ✅ полностью рабочая игра для двух игроков

---

## 📝 О проекте

Это классическая игра **«Крестики-нолики»** с графическим интерфейсом, написанная на **Java** с использованием **JavaFX** и **Maven**.

Игра предназначена для двух игроков на одном устройстве. Вы по очереди ставите X и O на поле 3×3. Побеждает тот, кто первым выстроит три своих знака в ряд (по горизонтали, вертикали или диагонали).

**Что умеет игра:**
- 🎮 Игра для двух игроков (человек против человека).
- 🖥️ Чистый и понятный интерфейс на JavaFX.
- 🧠 Автоматическая проверка победы и ничьей.
- 💬 Всплывающие уведомления с результатом партии.
- 🚫 Блокировка поля после окончания игры.

## 💻 Среда разработки и запуск

Проект создавался и запускается в среде **IntelliJ IDEA** (Community Edition) с использованием **JavaFX** и Maven.

### Как собрать проект у себя (для новичков)
1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/kepanasyuke/fumiklea.git
   cd fumiklea/tic_tac_toe_java/tic_tac_toe_svoy_cod
   ```
2. **Откройте в IntelliJ IDEA:** Выберите `File → Open...` и укажите путь к папке `tic_tac_toe_svoy_cod`. IDEA должна автоматически распознать Maven-проект.
3. **Настройте JavaFX (если требуется):** Убедитесь, что в `File → Settings → Languages & Frameworks → JavaFX` указан путь к Scene Builder (обычно IDEA настраивает это сама).
4. **Запустите игру:** Откройте класс `HelloApplication` и нажмите зеленую стрелку рядом с методом `main()`.

### Альтернативный запуск из командной строки
```bash
./mvnw clean javafx:run      # для macOS/Linux
.\mvnw.cmd clean javafx:run   # для Windows
```

## 🎮 Игровой процесс – как играть

1. Открывается окно с заголовком **«Tic Tac Toe!»**.
2. На поле 3×3 все клетки пусты.
3. Первым ходит **Игрок X**.
4. Кликайте по любой пустой клетке – туда встанет ваш знак.
5. Ход переходит к другому игроку.
6. Игра автоматически определяет:
   - **Победу** – если три одинаковых знака в ряд. Появится поздравление.
   - **Ничью** – если поле заполнено, но никто не выиграл.
7. После окончания игры все клетки блокируются.

> 💡 **Совет:** Хотите сыграть ещё раз? Пока что нужно перезапустить программу. В будущих версиях обязательно появится кнопка «Новая игра».

## 🔧 Детали реализации (для любопытных)

**Хранение поля:**
```java
private char[][] field = new char[3][3];
```
Изначально все ячейки содержат символ ` ` (пробел).

**Обработка клика:**
- По нажатию на кнопку вычисляются её координаты в `GridPane` (`getRowIndex`, `getColumnIndex`).
- Если ячейка пуста и игра активна – ставится `currentPlayer`.
- После этого проверяется победа или ничья.
- Если никто не выиграл – `currentPlayer` меняется на другого.

**Проверка победы:**
- Метод `checkWin()` сканирует 3 строки, 3 столбца и 2 диагонали.
- Как только находит три одинаковых непустых символа – возвращает `true`.

**Ничья:**
- `isDraw()` возвращает `true`, если все ячейки не пусты **и** при этом не было победы.

## 📖 Немного об игре «Крестики-нолики» (то, чего нет в других README)

Вы знали, что:

- **Древние корни** – археологи находят похожие игры на досках в Древнем Египте (около 1300 лет до н.э.). Правда, правила до нас дошли не полностью.
- **Полный перебор** – в классических крестиках-ноликах на поле 3×3 всего **255 168** возможных партий, если учитывать порядок ходов. Если не учитывать симметрию – всего 138 уникальных финальных позиций.
- **Ничья при идеальной игре** – если оба игрока не ошибаются, игра всегда заканчивается вничью. Первый игрок может гарантировать себе ничью, а второй – не проиграть.
- **Стратегия для новичков** – самый простой способ не проиграть: занимай центр, а потом углы. Центр даёт больше всего линий (4).
- **Компьютерный AI** – самый простой бот для этой игры пишется на 10-15 строчках кода (рандомные ходы). А идеальный бот (алгоритм минимакс) – уже в 50-100 строк. Некоторые студенты пишут его в качестве учебного задания на втором курсе.
- **В 2022 году** вышел научный труд, где крестики-нолики использовали для обучения нейросетей с подкреплением. Игра настолько проста, что идеально подходит как «полигон» для AI.

- игрового AI,
- событийно-ориентированного программирования (клики мыши),
- работы с графическим интерфейсом.
