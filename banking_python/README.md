# Корпоративный монитор комплаенса и антифрод-аналитики (SIEM Enterprise)

## О проекте
Проект реализован как интерактивный дашборд на Streamlit для анализа банковских транзакций и выявления мошеннических операций. Он загружает данные из файла `banking_transactions.csv`, строит метрики, визуализации и автоматически генерирует тексты отчетов для службы безопасности и регуляторов.

## Что делает приложение
* Загружает и нормализует CSV с транзакциями.
* Строит KPI: общее число операций, число мошеннических инцидентов, долю фрода, сумму транзакций.
* Позволяет фильтровать данные по риску устройства, методу оплаты и уровню аутентификации.
* Моделирует атаки и позволяет симулировать небезопасные транзакции.
* Демонстрирует распределение транзакций, корреляцию риска/попыток входа и уязвимости каналов оплаты.
* Показывает кнопку аналитики для расширенных инструментов работы с таблицей и сохранения пресетов.
* Автоматически генерирует тексты отчетов для внутреннего рапорта СБ, доклада ЦБ РФ и отчета 115-ФЗ.

## Что ещё можно добавить на кнопку
В текущем приложении уже есть кнопка симуляции атаки и стандартный экспорт логов. Хорошие дополнения:
* Экспорт текущего среза таблицы в CSV/XLSX.
* Сохранение выбранной фильтрации как готового пресета для повторного анализа.
* Сохранение отчета не только в TXT, но и в PDF или Excel.
* Кнопку "Показать детали по транзакции" для выбранной строки, чтобы сразу увидеть профиль риска конкретного логина.
* Кнопку "Отправить отчет в SIEM" или "Синхронизировать с BI", которая передавала бы текущие метрики в систему мониторинга.

## Какой инструмент аналитики лучше использовать для таблиц
Для работы с таблицами в интерфейсе удобнее всего использовать:
* `st_aggrid` + Streamlit — интерактивная таблица с сортировкой, фильтрацией, выделением и экспортом.
* `pandas` + `plotly` — для быстрого построения сводных таблиц, графиков и кросс-матриц.

Если нужна отдельная аналитическая платформа, можно подключить данные к:
* Metabase
* Apache Superset
* Power BI
* Grafana

Но в рамках этого проекта лучшим вариантом будет оставить Streamlit и добавить `st_aggrid`, чтобы рабочее место аналитика сразу давало интерактивный анализ таблиц.

## Аналитики и их задачи с этой информацией
Аналитики используют такие данные, чтобы:
* искать аномалии в платежном потоке и выявлять подозрительные векторы.
* обнаруживать слабые места в методах аутентификации (`Password` / `None` / `SMS OTP`).
* анализировать каналы эквайринга и выявлять «горячие» шлюзы с высоким уровнем фрода.
* рассчитывать матрицу ошибок (ложные тревоги vs. пропущенный фрод) для оптимизации правил SIEM.
* готовить регуляторные отчёты для ЦБ РФ, Финмониторинга и внутренней службы безопасности.
* проверять данные через криминалистические методы, такие как закон Бенфорда.

## Запуск приложения
1. Откройте терминал в папке `banking_python`.
2. Создайте виртуальное окружение:
```bash
python3 -m venv venv
source venv/bin/activate
```
3. Установите зависимости:
```bash
pip install -r requirements.txt
```
4. Запустите приложение:
```bash
streamlit run app.py
```
5. Откройте браузер по адресу:
```text
http://localhost:8501
```

### Примечания
* Если `banking_transactions.csv` отсутствует, приложение сгенерирует синтетический тренировочный набор данных и продолжит работу.
* Для расширенного анализа таблиц можно дополнительно установить:
```bash
pip install streamlit-aggrid
```
* Если нужен формат экспорта в Excel, добавьте:
```bash
pip install openpyxl
```
