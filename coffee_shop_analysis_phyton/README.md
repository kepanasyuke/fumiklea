# 📊 Анализ продаж кофейни "Уютный уголок"

Полностью автоматизированная система для анализа и визуализации данных продаж кофейни с генерацией красивого PDF-отчёта.

## 🎯 Особенности

- ✅ **Автоматическая генерация данных** — синтетический датасет с 10,000+ транзакций
- 📈 **Многоуровневый анализ** — почасовые, дневные и временные тренды
- 🎨 **Красивая визуализация** — 5 типов графиков (столбчатые, круговые, тепловые карты)
- 📄 **Профессиональный PDF-отчёт** — сгенерирован через Jinja2 + WeasyPrint
- 🔄 **Полная модульность** — легко расширяется и адаптируется
- 🛡️ **Обработка ошибок** — try/except на всех критических участках

## 🚀 Быстрый старт

### 1️⃣ Установка зависимостей

```bash
# Создайте виртуальное окружение
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# или .venv\Scripts\activate на Windows

# Установите зависимости
pip install -r requirements.txt
```

### 2️⃣ Подготовка шрифтов (опционально)

Для гарантированной поддержки кириллицы:

```bash
# Linux/macOS
make fonts

# Windows (вручную)
# 1. Скачайте DejaVuSans.ttf
# 2. Поместите в папку fonts/
```

### 3️⃣ Запуск генерации отчёта

```bash
# С синтетическими данными (по умолчанию 10,000 записей)
python run.py

# С собственными синтетическими данными (50,000 записей)
python run.py --records 50000

# С реальными данными из CSV файла
python run.py --csv data/my_sales.csv

# Или через Makefile
make report

# Очистка (удаление PDF)
make clean
```

**Результат**: файл `reports/coffee_shop_report.pdf`

## 📊 Что входит в отчёт

### 1. Ключевые метрики
- **Пиковый час** — время с максимальной выручкой
- **Лучший день недели** — день с наибольшей суммой продаж
- **Топ-категория** — основной источник выручки
- **Средний чек** — по времени суток (утро, день, вечер)

### 2. Визуализации (УЛУЧШЕНЫ в v2.1)
- 📊 **Выручка по часам** — крупная столбчатая диаграмма с значениями
- 📅 **Выручка по дням недели** — рабочие дни vs выходные (синий vs оранжевый)
- 🥧 **Доля категорий** — красивая круговая диаграмма с процентами
- 💰 **Средний чек по времени суток** — сравнение утро/день/вечер с символами
- 🔥 **Тепловая карта** — полная матрица (день недели × час) с цветовой шкалой

### 3. Рекомендации
- Оптимизация расписания персонала в пиковые часы
- Стратегии повышения среднего чека
- Управление ассортиментом

### 4. Оформление (УЛУЧШЕНО в v2.1)
- ✨ Градиентные заголовки с цветной полосой
- 📱 Отзывчивый дизайн с 2-колончным макетом
- 🎨 Тени и анимации при наведении
- 🖨️ Оптимизирован для печати

## 💾 Работа с реальными данными

### Формат CSV файла

Ваш CSV должен содержать **минимум** эти колонки:
```
date,hour,category,revenue
2024-06-01,08,Кофе,220
2024-06-01,09,Кофе,500
2024-06-01,12,Сэндвичи,290
```

**Полный формат** (рекомендуется):
```
date,hour,category,item,price,quantity,revenue,payment_method
2024-06-01,08,Кофе,Капучино,220,1,220,Карта
```

### Пример загрузки реальных данных

```bash
# Загрузить данные из CSV файла
python run.py --csv data/coffee_sales_2024.csv

# Проверить структуру данных
python -c "
from src.analysis import load_data_from_csv
df = load_data_from_csv('data/coffee_sales_2024.csv')
print(df.head())
print(f'Всего записей: {len(df)}')
"
```

### Как подготовить свой CSV файл

1. **Убедитесь, что есть колонки**: `date`, `hour`, `category`, `revenue`
2. **Формат даты**: `YYYY-MM-DD` (например, `2024-06-15`)
3. **Час**: целое число 0-23
4. **Категория**: одна из (Кофе, Чай, Выпечка, Сэндвичи) или своя
5. **Выручка**: число в рублях

Пример своего CSV:
```csv
date,hour,category,revenue
2024-01-01,08,Кофе,220
2024-01-01,09,Кофе,500
2024-01-01,12,Сэндвичи,290
```

## ⚠️ Возможные проблемы и решения

### Ошибка: `OSError: no library called "cairo-2" was found`

**Windows:**
```powershell
# Установите GTK+ 3 через winget
winget install Gnome.Project.Gtk3
# Или вручную: скачайте с https://github.com/tschoonj/GTK-for-Windows-Runtime-Environment-Installer/releases
```

**Linux (Debian/Ubuntu):**
```bash
sudo apt-get update && sudo apt-get install libpango-1.0-0 libcairo2 libgdk-pixbuf2.0-0
```

**macOS:**
```bash
brew install cairo pango gdk-pixbuf libffi
```

### Кириллица отображается кракозябрами

- Скопируйте `DejaVuSans.ttf` в папку `fonts/`
- Или используйте системный шрифт (автоматическая подстановка в `setup_russian_font()`)

### Графики не вставляются в PDF

- Проверьте, что все графики успешно конвертированы в base64
- Убедитесь, что директория `reports/` доступна для записи
