## 📝 О проекте

Проект **decimal_C** — это учебная реализация библиотеки `s21_decimal.h` на языке C (стандарт C11). Она добавляет в язык возможность работы с пользовательским типом данных `decimal`, который критически важен для финансовых и бухгалтерских расчётов, где недопустимы ошибки округления, свойственные числам с плавающей точкой (float/double).

Тип `decimal` представляет собой 96-битное целое число со знаком, дополненное специальным масштабным коэффициентом (степенью десяти), что позволяет хранить до 28 значащих десятичных цифр и работать с числами в огромном диапазоне — примерно от ±7.9×10⁻²⁸ до ±7.9×10²⁸.

Проект состоит из заголовочного файла `s21_decimal.h` с объявлениями структур и функций, файла `s21_decimal.c` с их реализацией, набора тестов (`tests.c`) и Makefile для сборки.

## 💻 Сборка и запуск

### Подготовка к работе

1. **Убедитесь, что установлен компилятор gcc (версия с поддержкой C11).**
2. **Перейдите в директорию проекта:**
   ```bash
   cd fumiklea/decimal_C/src
   ```
3. **Соберите проект с помощью Makefile:**
   ```bash
   make all
   ```
   Эта команда создаст статическую библиотеку `s21_decimal.a` на основе файлов `s21_decimal.c` и `s21_decimal.h`.

### Запуск тестов

Для проверки работоспособности всех функций используйте команду:
```bash
make test
```
Она соберёт и запустит программу `tests.c`, которая проверит корректность работы арифметических операций и функций преобразования. Для очистки артефактов сборки выполните `make clean`.

Чтобы использовать библиотеку в собственном проекте:


   ```bash
   gcc -std=c11 my_program.c -L. -l:s21_decimal.a -o my_program
   ```


## ⚙️ Архитектура и ключевые функции

### Структура `s21_decimal`

Структура `s21_decimal` реализована как массив из четырёх 32-битных целых чисел (total 128 бит):
- `bits[0]`, `bits[1]`, `bits[2]` — хранят 96-битную мантиссу числа.
- `bits[3]` — хранит служебную информацию: знак числа (31-й бит) и степень масштаба (биты с 16 по 23, значение от 0 до 28).

Такой подход обеспечивает максимальную точность представления десятичных чисел.

### Реализованные операции

Библиотека поддерживает набор операций, стандартный для финансовых библиотек:

| Категория | Функции |
|-----------|---------|
| **Арифметические** | `s21_add`, `s21_sub`, `s21_mul`, `s21_div`, `s21_mod` |
| **Сравнения** | `s21_is_less`, `s21_is_less_or_equal`, `s21_is_greater`, `s21_is_greater_or_equal`, `s21_is_equal`, `s21_is_not_equal` |
| **Преобразования** | `s21_from_int_to_decimal`, `s21_from_decimal_to_int`, `s21_from_float_to_decimal`, `s21_from_decimal_to_float` |
| **Округление** | `s21_floor`, `s21_round`, `s21_truncate`, `s21_negate` |

Все функции следуют правилам стандарта IEEE 754 для десятичной арифметики: при операциях корректно обрабатываются переполнение (overflow), потеря значимости (underflow) и деление на ноль.

## 💡 Почему decimal, а не float/double?

Числа с плавающей точкой (`float`, `double`) хранятся в двоичном виде, поэтому многие десятичные дроби (например, 0.1) не могут быть представлены точно. В финансовых системах это приводит к накоплению ошибок, что недопустимо. Тип `decimal` хранит числа в десятичном виде, гарантируя точность при работе с денежными суммами, процентами и другими величинами.
