# Конвертер Валют

Простое консольное приложение для конвертации валют с использованием внешнего API.

## Описание

Приложение позволяет пользователю ввести сумму, код исходной валюты и код целевой валюты, после чего получает актуальный курс обмена из API и выводит результат конвертации.

## Установка и Запуск

### Требования
- Python 3.8+
- pip

### Установка зависимостей
```bash
pip install -r requirements.txt
```

### Запуск
Из папки `currency_converter/src/`:
```bash
make run
```

Или напрямую:
```bash
PYTHONPATH=/workspaces/fumiklea python -m currency_converter.src.main
```

## Использование
Программа запрашивает:
1. Сумму для конвертации (число).
2. Код исходной валюты (например, USD).
3. Код целевой валюты (например, RUB).

Выводит результат в формате: `100.00 USD = 8167.00 RUB (курс: 1 USD = 81.6700 RUB)`

## Архитектура и Решения

### Структура проекта
```
currency_converter/
├── src/
│   ├── __init__.py
│   ├── api_client.py      # Получение курсов из API
│   ├── converter.py       # Логика конвертации
│   ├── exceptions.py      # Кастомные исключения
│   ├── main.py            # Точка входа, CLI
│   └── utils.py           # Валидация и форматирование
├── tests/
│   ├── __init__.py
│   ├── test_api_client.py
│   ├── test_converter.py
│   └── test_utils.py
├── __init__.py
├── Makefile               # Автоматизация задач
└── requirements.txt       # Зависимости
```

### Почему такая структура?
- **Модульность**: Код разделён на небольшие модули по ответственности (Single Responsibility Principle). Это упрощает тестирование, поддержку и расширение.
- **Исключения**: Используются кастомные исключения (`APIError`, `CurrencyNotFoundError`, `InvalidAmountError`) для чёткой обработки ошибок. Они наследуются от `Exception` и позволяют передавать детализированные сообщения.
- **Валидация**: Входные данные проверяются на корректность (положительная сумма, существующие валюты). Это предотвращает некорректные расчёты и улучшает UX.
- **Округление**: Результаты конвертации округляются до 2 десятичных знаков, так как валюты обычно имеют 2 знака после запятой.
- **API**: Выбран `exchangerate-api.com` как бесплатный и надёжный источник курсов. Обработка ошибок API (таймауты, неверный JSON, отсутствие валюты).
- **Тесты**: Полное покрытие с unittest и mock'ами для изоляции внешних зависимостей. Тесты проверяют как успешные сценарии, так и исключения.
- **Makefile**: Автоматизирует установку, тестирование, линтинг и запуск. Упрощает разработку.
- **Относительные импорты**: Используются для поддержки пакетной структуры, что позволяет запускать модули как пакеты.

### Решения для код-ревью
- **Обработка ошибок**: Вместо возврата кодов ошибок используются исключения для чистоты кода. Все исключения ловятся в `main()` для пользовательского вывода.
- **Типизация**: Добавлены type hints для улучшения читаемости и поддержки IDE.
- **Docstrings**: Все функции имеют docstrings с описанием, аргументов, возвратов и исключений (по PEP 257).
- **Линтинг**: Код соответствует PEP 8, проверяется flake8.
- **Тестирование**: 16 тестов покрывают все модули. Используются mock'и для API-запросов, чтобы тесты были быстрыми и независимыми.
- **Без зависимостей**: Минимальные зависимости (requests для HTTP), чтобы проект был лёгким.
- **CLI**: Простой интерфейс без фреймворков, фокус на логике.
- **Курс на единицу**: В выводе показывается курс "1 base = rate target" для ясности.
